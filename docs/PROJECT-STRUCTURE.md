# Структура проекта

## Обзор

Проект организован по принципу функциональной группировки компонентов, утилит и стилей для обеспечения масштабируемости и поддерживаемости.

## Принципы организации

### 1. Компоненты по функциональности (features/)

Компоненты сгруппированы по бизнес-логике:
- `portfolio/` - все компоненты, связанные с портфолио
- `hero/` - главная секция сайта
- `services/` - секция услуг
- `team/` - секция команды
- `pricing/` - секция ценообразования

**Преимущества:**
- Легко найти все компоненты, связанные с одной функциональностью
- Упрощает рефакторинг и удаление функциональности
- Улучшает переиспользование кода

### 2. Layout компоненты

Компоненты, которые используются на всех или большинстве страниц:
- `Navigation.tsx` - навигация
- `Footer.tsx` - футер
- `Logo.tsx` - логотип

### 3. Страничные компоненты (pages/)

Компоненты, которые представляют целые страницы:
- Используются в App Router как страницы
- Содержат логику страницы и композицию компонентов

### 4. Общие компоненты (shared/)

Компоненты, используемые в разных частях приложения:
- `CTASection.tsx` - секция призыва к действию
- `ContactModal.tsx` - модальное окно контактов
- `SEO.tsx` - SEO метаданные

### 5. UI компоненты (ui/)

Переиспользуемые UI компоненты из shadcn/ui:
- Базовые компоненты интерфейса
- Не содержат бизнес-логику
- Используются во всех частях приложения

## Утилиты

### performance.ts
Оптимизация производительности:
- Определение возможностей устройства
- Конфигурация анимаций
- Throttle/debounce функции

### motionConfig.ts
Конфигурация анимаций:
- Варианты анимаций (fadeIn, fadeInUp, etc.)
- Оптимизированные переходы
- Поддержка reduced motion

### analytics.ts
Аналитика:
- Трекинг событий
- Интеграция с аналитическими сервисами

### constants.ts
Константы приложения:
- URL-адреса
- Текстовые константы
- Настройки

## Стили

### globals.css
Глобальные стили:
- CSS переменные для темизации
- Анимации
- Утилиты для производительности
- Медиа-запросы для мобильных устройств

## Именование файлов

- **Компоненты**: PascalCase (например, `Portfolio.tsx`)
- **Утилиты**: camelCase (например, `motionConfig.ts`)
- **Стили**: kebab-case (например, `globals.css`)
- **Константы**: UPPER_SNAKE_CASE (внутри файлов)

## Импорты

Используются абсолютные импорты через `@/`:
```typescript
import { Portfolio } from "@/src/components/Portfolio";
import { getAnimationConfig } from "@/src/utils/performance";
```

## Масштабируемость

Структура спроектирована для легкого масштабирования:
1. Новые функции добавляются в `features/`
2. Новые страницы добавляются в `pages/`
3. Общие компоненты выносятся в `shared/`
4. Утилиты группируются по назначению в `utils/`

## Миграция

Текущая структура постепенно мигрирует к описанной выше:
- Существующие компоненты остаются на своих местах для обратной совместимости
- Новые компоненты создаются в новой структуре
- Постепенная миграция существующих компонентов
